# Security & Quality Audit — 2026-03-26

Findings from senior code review. All items resolved.

## Fixed (first session)

- **[CRITICAL C2]** `RotateKey` now copies `newKey` before passing to C and
  zeroes the copy on return. Prevents caller mutation during the C call.
- **[HIGH H3]** `rows.Close()` now acquires `conn.mu` before `sqlite3_finalize`.
- **[HIGH H4]** `stmt.Close()` now acquires `conn.mu` before `sqlite3_finalize`.
- **[HIGH H1]** Removed `newConnector()` — was inconsistent with `Open()` for
  WAL-on-memory. Test uses `buildConnector()` (extracted from `Open()`).
- **[LOW L1]** Simplified redundant `retErr` in `RotateKey`.
- **[LOW L3]** `TestConnectorKeyZeroedOnClose` now verifies bytes are zeroed,
  not just that the slice is nil.

## Fixed (second session — deferred items)

- **[CRITICAL C1]** Added doc comment on `connector.Close()` and package doc
  explaining that key zeroing is best-effort due to GC heap compaction.
- **[HIGH H2]** `RotateKey` now updates the connector's key in place after
  successful rekey. Added `parent *connector` back-reference on `conn`.
  Callers no longer need to close and re-open after rotation.
- **[MEDIUM M1]** Added `runtime.KeepAlive(v)` after `securedb_bind_blob` call
  in `bindArgs` for `[]byte` arguments (defense-in-depth).
- **[MEDIUM M2]** Added comment on `isMemoryPath` explaining why only `:memory:`
  is checked (no `SQLITE_OPEN_URI` flag) and what to update if URI support is added.
- **[MEDIUM M3]** Documented `ExecContext` multi-statement behavior difference:
  fast path (sqlite3_exec) executes all statements, slow path (prepare) only first.
- **[MEDIUM M6]** Replaced hardcoded key in `Available()` with `crypto/rand`
  generated 32-byte key.
- **[LOW L2]** Documented `BeginTx` `ReadOnly` limitation: `BEGIN DEFERRED`
  does not enforce read-only semantics in SQLite.
- **[LOW L4]** Documented Darwin CGO path limitation in package doc: Apple
  Silicon path is hardcoded, Intel Macs need manual `CGO_CFLAGS`/`CGO_LDFLAGS`.

## Fixed (second session — post-audit findings)

- **[HIGH]** `RotateKey` connector key update now copies from `keyCopy`
  (defensive copy) instead of `newKey` (caller's slice), closing TOCTOU window.
- **[HIGH]** Added `sync.RWMutex` (`keyMu`) to `connector` protecting the key
  field. `Connect()` takes read lock; `RotateKey` and `Close()` take write lock.
  Prevents data races even with `MaxOpenConns > 1`.
- **[MEDIUM]** Validated `EncryptionMode` in `buildConnector` — unknown values
  now return an explicit error.
- **[MEDIUM]** Validated `BusyTimeout >= 0` in `buildConnector` — negative
  durations now return an error.
